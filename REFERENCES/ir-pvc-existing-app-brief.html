IR-PVC Reverse Engineering

Existing PVC Application Architecture

IR-PVC is a vertical SaaS platform for Indian Railway contractors that automates Price Variation Clause calculations using contract setup data, bill classifications, and a centralized historical index database covering labour, fuel, steel, and material price series.

Contract lifecycle management PVC calculation workflow Wallet-gated computation Zone-specific index usage PDF engineering reports
9+ screens across contract, bill, and report workflow
3 core moats: index pipeline, rules engine, wallet architecture
Manual classification mapping remains a dominant operator burden
The application does not appear to own the full contractor billing workflow yet. It primarily standardizes the final PVC computation layer after users bring in already-prepared bill values.

Core Product Thesis

Replace spreadsheet-based PVC calculations with a controlled workflow that starts at contract setup, carries extension and classification context into running bills, and outputs a standardized engineering report.

The current product still leaves upstream quantity generation outside the system. That limits automation and keeps Excel in the loop.

Formula Analysis

PVC Engine Inference

Calculation Pipeline

Observed Formula Logic

1. Base month = usually one month prior to date of opening 2. Work classification carries fixed component percentage weights 3. Current quarter index is resolved from measurement date 4. Base index and current index are fetched from historical data store 5. PVC formula: (Amount * Component %) * ((Current Quarter Index - Base Index) / Base Index) 6. Steel logic uses JPC zone-specific retail market rates 7. Fuel logic uses railway zone / diesel basis selection 8. Extension status can alter eligibility or capping behavior
Deterministic core
the calculation appears rules-based, not probabilistic
Strong reference-data dependency
index accuracy is central to correctness
Classification-driven percentages
weights appear keyed by standardized railway work categories
Penalty-aware eligibility
extensions can modify allowable PVC treatment

Modernization Path

Move upstream
ingest MB or Excel quantity sheets to generate bill amounts before PVC.
Reduce mapping friction
persist reusable contract-specific classification mappings and auto-apply them on later bills.
Shift wallet gating earlier
expose cost and credit needs before final submission step.
Document-assisted autofill
pull agreement numbers and descriptions directly from uploaded tender/agreement files.
The largest product opportunity is to stop being only a PVC calculator and become the contractor billing operating system that produces PVC as a natural downstream output.

Primary Moat

A clean, complete, and continuously updated RBI / JPC index pipeline is the hardest asset to replicate reliably.

Rules Moat

The deterministic GCC-aware rule engine becomes more valuable as it accumulates extension, penalty, and classification edge cases.

Commercial Control

The wallet and tenant model monetizes computation cleanly, but it should be paired with better workflow timing to reduce user frustration.

This brief summarizes the existing IR-PVC application as a reverse-engineered SaaS product: a contract-driven, index-backed PVC calculation platform with wallet monetization, guided bill processing, and standardized report generation.
